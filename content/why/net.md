## tcp

- 为什么 TCP 建立连接需要三次握手
- 为什么 DNS 使用 UDP 协议
- 为什么 TCP 协议有性能问题
- 为什么 UDP 头只有 8 个字节
- 为什么 TCP/IP 协议会拆分数据
- 为什么流媒体直播的延迟很高
- 为什么 HTTPS 需要 7 次握手以及 9 倍时延
- 为什么 TCP 协议有粘包问题
- 为什么 TCP 协议有 TIME_WAIT 状态
- 为什么 MAC 地址不需要全球唯一
- 为什么 IPv6 难以取代 IPv4
- 为什么集群需要 Overlay 网络
- 为什么 TCP 拥塞控制很重要
- 为什么 TCP 有保活机制
- 为什么 TCP 有快重传和快恢复
- 为什么 TCP 有延迟确认
- 为什么 TCP 有滑动窗口机制

## udp

- 为什么 UDP 没有连接状态
- 为什么 UDP 可以一对多通信
- 为什么 UDP 适合实时应用
- 为什么 UDP 比 TCP 简单
- 为什么 UDP 需要应用层可靠性保证
- 为什么 UDP 有校验和
- 为什么 UDP 可以广播和组播
- 为什么 UDP 适合游戏应用
- 为什么 UDP 有包丢失问题

## http

- 为什么 HTTP 使用明文传输
- 为什么 HTTP/1.1 有队头阻塞问题
- 为什么 HTTP 需要 Cookie 和 Session
- 为什么 HTTPS 比 HTTP 慢
- 为什么 HTTP/2 使用多路复用
- 为什么 HTTP 有缓存机制
- 为什么 HTTP 有状态码
- 为什么 HTTP 有压缩传输
- 为什么 HTTP 有跨域限制
- 为什么 HTTP 使用长连接
- 为什么 HTTP/3 使用 QUIC 协议

## network

- 为什么网络需要分层架构
- 为什么网络有 OSI 七层模型
- 为什么网络需要路由器
- 为什么网络需要交换机
- 为什么网络需要 NAT
- 为什么网络需要防火墙
- 为什么网络有负载均衡
- 为什么网络有 QoS 机制
- 为什么网络有 MTU 限制
- 为什么网络有 ARP 协议
- 为什么网络有 ICMP 协议
- 为什么网络需要 DNS 解析
