# 创建新博客文章

## 功能说明
使用此命令创建新的技术博客文章，自动生成符合Hugo格式的markdown文件，包含正确的front matter和文章结构。

## 使用步骤

### 1. 收集文章信息
请用户提供以下信息：
- **文章标题**: 清晰描述文章内容的标题
- **技术分类**: 选择文章所属的技术分类（go, devops, kubernetes, micro, container, engineer, vue, llm, coding）
- **文章类型**: 独立文章或系列文章
- **如果是系列文章**: 提供系列名称和编号
- **主要标签**: 相关的技术标签
- **文章大纲**: 简要的文章内容大纲

### 2. 确定文件位置
根据技术分类确定正确的目录路径：
- Go编程: `content/post/go/{子分类}/`
- DevOps: `content/post/devops/`
- Kubernetes: `content/post/kubernetes/`
- 其他分类: `content/post/{分类名}/`

### 3. 生成Front Matter
根据提供的信息生成标准的Hugo front matter：
```yaml
---
title: "用户提供的标题"
date: 当前日期时间
lastmod: 当前日期时间
draft: true
tags: ["标签1", "标签2"]
categories: ["分类名"]
author: "yesplease"
---
```

### 4. 创建文章结构
根据文章类型和大纲创建文章结构：
1. **独立文章**: 标准的技术文章结构
2. **系列文章**: 包含系列导航的标准结构

### 5. 提供写作指导
根据文章主题提供具体的写作建议：
- 技术要点解释
- 代码示例建议
- 最佳实践指南
- 常见问题解答

### 6. 预览和编辑
创建文件后，提供预览并允许用户编辑完善。

## 示例场景

```
用户输入:
- 标题: "Go并发编程入门"
- 分类: go
- 子分类: base
- 类型: 系列文章
- 系列名称: "go-concurrent"
- 编号: 1
- 标签: ["go", "并发", "goroutine"]
- 大纲: "介绍goroutine基础、channel使用、select语句"

处理流程:
1. 确定路径: content/post/go/base/
2. 文件名: series-go-concurrent-1.md
3. 生成front matter
4. 创建系列导航
5. 提供Go并发写作指导
```

- 标签: ["kubernetes", "部署", "devops"]

处理流程:

## 最佳实践

### 技术文章结构建议
1. **引言**: 问题陈述和重要性
2. **核心概念**: 技术原理解释
3. **基础用法**: 入门示例
4. **高级特性**: 进阶功能
5. **实际应用**: 项目案例
6. **最佳实践**: 经验总结
7. **总结**: 内容回顾

### 代码示例要求
1. **完整性**: 提供可运行的代码
2. **注释**: 关键代码添加注释
3. **解释**: 说明代码作用和原理
4. **最佳实践**: 遵循语言规范
5. **安全性**: 避免安全漏洞

### 系列文章规范
1. **命名**: series-{主题}-{编号}.md
2. **导航**: 每篇文章包含系列导航
3. **连续性**: 内容循序渐进
4. **完整性**: 覆盖主题全面
5. **独立性**: 每篇文章可独立阅读

## 输出要求

### 文件创建
1. 在正确目录创建markdown文件
2. 使用UTF-8编码
3. 遵循项目代码风格
4. 文件名使用kebab-case

### 内容质量
1. 技术准确性
2. 结构清晰性
3. 示例实用性
4. 可读性良好

### 交互指导
1. 提供编辑建议
2. 允许内容调整
3. 支持预览功能
4. 可重新生成

## 错误处理

### 无效输入处理
- 分类不存在: 提供可用分类列表
- 路径错误: 检查目录结构
- 格式错误: 纠正输入格式
- 内容冲突: 检查文件是否已存在

### 恢复机制
- 备份原始文件
- 提供撤销选项
- 保存中间状态
- 记录操作日志
