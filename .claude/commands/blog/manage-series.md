# 管理博客文章系列

## 功能说明
使用此命令管理技术博客的系列文章，包括创建新系列、添加系列文章、更新系列导航和维护系列完整性。

## 使用步骤

### 1. 系列状态检查
首先检查现有的系列文章状态：
```bash
# 查找所有系列文章
find content/post -name "series-*.md" | sort

# 分析系列结构
检查系列命名规范、编号连续性和内容完整性
```

### 2. 系列操作选择
根据用户需求选择操作类型：
1. **创建新系列**: 开始全新的技术系列
2. **添加系列文章**: 为现有系列添加新文章
3. **更新系列导航**: 更新系列文章的导航链接
4. **重组系列**: 调整系列结构或内容
5. **归档系列**: 完成系列并归档

### 3. 系列信息收集
根据操作类型收集必要信息：

#### 创建新系列
- **系列主题**: 技术主题名称
- **目标分类**: 技术分类目录
- **预计篇幅**: 计划文章数量
- **系列大纲**: 各篇文章主题规划
- **目标读者**: 预期的读者群体

#### 添加系列文章
- **系列名称**: 现有系列名称
- **文章编号**: 新文章编号
- **文章主题**: 本篇文章主题
- **内容大纲**: 具体内容规划
- **关联文章**: 与前后文章的关联

#### 更新系列导航
- **系列名称**: 要更新的系列
- **导航范围**: 需要更新的文章范围
- **链接格式**: 导航链接格式要求
- **额外信息**: 需要添加的系列信息

### 4. 文件操作执行
根据操作类型执行相应的文件操作：

#### 创建新系列
1. 创建系列目录结构
2. 生成系列规划文档
3. 创建第一篇系列文章
4. 设置系列导航模板

#### 添加系列文章
1. 检查系列完整性
2. 确定正确编号
3. 创建新文章文件
4. 更新前后文章导航
5. 维护系列索引

#### 更新系列导航
1. 批量更新系列导航
2. 验证链接正确性
3. 更新系列索引页
4. 检查导航一致性

### 5. 系列质量检查
完成操作后检查系列质量：
1. **命名规范**: 符合series-{topic}-{number}.md格式
2. **编号连续性**: 编号连续无跳跃
3. **内容连贯性**: 文章间内容衔接自然
4. **导航完整性**: 所有文章正确链接
5. **格式一致性**: 统一的内容格式

### 6. 提供管理报告
生成系列管理报告：
- 操作总结
- 当前状态
- 问题发现
- 后续建议

## 系列管理规范

### 命名规范
1. **文件命名**: `series-{topic}-{number}.md`
2. **主题命名**: 使用kebab-case
3. **编号规则**: 从1开始连续编号
4. **目录结构**: 按技术分类组织

### 内容规范
1. **系列介绍**: 每篇包含系列说明
2. **导航标准**: 统一的导航格式
3. **内容连贯**: 前后文章内容衔接
4. **难度递进**: 从基础到进阶
5. **实践结合**: 理论结合实际案例

### 导航格式
```markdown
> 本系列文章索引：
> 1. [第一部分：基础概念](/post/category/series-topic-1/)
> 2. [第二部分：进阶应用](/post/category/series-topic-2/)
> 3. [第三部分：具体主题](/post/category/series-topic-3/) ← 当前文章
> 4. [第四部分：实战案例](/post/category/series-topic-4/)
```

### 系列索引
建议创建系列索引页：
```markdown
# 系列名称完整指南

## 系列概述
介绍系列目的和学习目标。

## 文章列表
1. [第一部分：基础概念](/post/category/series-topic-1/)
   - 内容简介
   - 学习要点
   - 实践练习

2. [第二部分：进阶应用](/post/category/series-topic-2/)
   - 内容简介
   - 学习要点
   - 实践练习

## 学习路径
建议的学习顺序和方法。

## 资源下载
相关的代码和资源。
```

# 更新系列导航

update_series_navigation() {
    local category="$1"
    local series="$2"
    local total="$3"

    for i in $(seq 1 $total); do
        file="content/post/$category/series-$series-$i.md"
        if [ -f "$file" ]; then
            # 生成导航内容
            nav_content="> 本系列文章索引：\n"

### 系列统计报告
```bash
#!/bin/bash
# 生成系列统计报告

generate_series_report() {
    echo "系列文章统计报告"
    echo "=================="
    echo ""

    # 按分类统计
    find content/post -type d | while read dir; do
        series_count=$(find "$dir" -name "series-*.md" | wc -l)
        if [ $series_count -gt 0 ]; then
            echo "分类: $(basename "$dir")"
            echo "系列数量: $series_count"
            find "$dir" -name "series-*.md" | sed 's/.*series-\([^-]*\)-[0-9]*\.md/\1/' | sort | uniq | while read series; do
                article_count=$(find "$dir" -name "series-$series-*.md" | wc -l)
                echo "  系列: $series, 文章数: $article_count"
            done
            echo ""
        fi
    done
}
```
